# Entity 模式

**目录：** [应用场景](#应用场景) · [常见模式](#常见模式) · [模式选择指南](#模式选择指南)

## 应用场景

### 模型-视图分离

使用实体将业务逻辑（模型）与 UI（视图）分离。

### 组件状态管理

使用实体管理复杂的组件状态。

### 跨实体通信

协调父子实体之间的状态。

### 异步操作与实体

管理异步状态更新。

### 后台任务协调

将后台任务与实体更新结合使用。

## 常见模式

### 1. 有状态组件

对维护内部状态的组件使用实体。

### 2. 共享状态

使用实体在多个组件之间共享状态。

### 3. 事件协调

使用实体协调组件之间的事件。

### 4. 异步状态管理

管理基于异步操作变化的状态。

### 5. 父子关系

使用弱引用管理层次关系。

### 6. 观察者模式

使用观察者响应实体状态变化。

### 7. 事件订阅

处理其他实体发送的事件。

### 8. 资源管理

使用适当的清理管理外部资源。

## 模式选择指南

| 需求 | 模式 | 复杂度 |
|------|------|--------|
| 带内部状态的组件 | 有状态组件 | 低 |
| 多个组件共享的状态 | 共享状态 | 低 |
| 协调组件之间的事件 | 事件协调 | 中 |
| 处理异步数据获取 | 异步状态管理 | 中 |
| 父子组件层次结构 | 父子关系 | 中 |
| 响应状态变化 | 观察者模式 | 中 |
| 处理自定义事件 | 事件订阅 | 中-高 |
| 管理外部资源 | 资源管理 | 高 |

选择满足需求的最简单模式。根据需要组合模式以处理复杂场景。
